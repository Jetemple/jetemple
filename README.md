### What I like working on

I'm a backend engineer who likes to tinker, build, and make tools I actually use.

Most of my work is around backend systems, infrastructure, and AI platforms, but the projects I enjoy most usually start with a personal annoyance: something slow, repetitive, fragile, or missing that I want to fix.

Recently I've been interested in local-first software, LLM tooling, agent infrastructure, speech-to-text, and developer tools.
